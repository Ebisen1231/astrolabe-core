# UI リサーチ: Notion系パターンの棚卸しと採否(M3.5設計資料)

作成: 2026-07-20。施主の「学習体験をNotionに近づけたい」を受けた機能リサーチ。
評価軸は3つ: **学習ループ(提案→選択→学習→記録→マップ)への寄与 / 実装コスト / 単独利用者への適合**。
Notionの模倣が目的ではなく、「読む・辿る・記録する」の摩擦を減らすパターンの選別が目的。

参照: Notion 3.4(2026-03)でサイドバー刷新(4タブ構成)・ページ内タブブロック・
ページアーカイブ・ダッシュボード等が追加。データベースのタブ表示カスタマイズは2026-04。

## 採否マトリクス

### Tier 1 — M3.5で採用

| パターン | Astrolabeでの形 | 根拠 |
|---|---|---|
| ライトテーマの読書面 | マップ以外を白地・墨色・行間1.75へ(承認済みモック) | 「観測は夜、学習は昼」。長文可読性 |
| ブロック構造のページ | トピック詳細: パンくず/チップ/コールアウト(なぜ今)/箇条書き/実践課題カード | 設計書§7の宿題(Notion風ブロック) |
| **実践課題の「タスク化」ボタン** | 詳細ページから既存 create_task API を1クリック呼び出し | 読む→やる→記録が画面内で閉じる。Notionにない優位 |
| サイドバー+ページツリー | 今日の報告 / 履歴(日付ツリー) / 概念 / タスク / チューター。折りたたみ可 | 上部タブは増設限界。Notionの背骨 |
| **概念ページ+バックリンク** | 概念クリック→専用ページ: summary・出典・**登場した報告/タスク/隣接概念の一覧** | edgesとeventsが既にあるので**データ追加ゼロ**で双方向リンクが立つ(Obsidianの核) |
| クイック検索(⌘K) | 概念・報告・タスクの横断検索。**ヒットなし→「チューターに聞く」ボタン** | 検索失敗=未知語との遭遇。M3完了条件をUIの一等地に置く |
| 数式レンダリング | KaTeX導入(UI依存+1、要提案)。学習コンテンツのO(n²)や式を組版 | LLM学習領域では数式が本文。可読性に直結 |
| タスクのビュー | リスト+状態/種別フィルタ。evidence付き完了は維持 | 現状の素朴一覧の自然な強化 |

### Tier 2 — M4以降・バックログ

- **ページ内タブブロック**(Notion 3.4) — 報告ページのトピック切替に使えるが、詳細ページ化で当面充足
- **toggleブロック(段階開示)** — 学習コンテンツの「深掘り層」。統合報告スキーマの拡張が前提なので復習間隔(M4)と同時に設計
- 履歴のカレンダービュー / お気に入り・ピン(学習キューはselectedイベントとの整理が先)
- アーカイブ表示(dismissed概念の可視化)
- ダッシュボード — Astrolabeでは星図マップが既にその役割

### Tier 3 — 採らない(理由つき)

- コメント・共有・権限 — 単独利用者設計に不要
- テンプレート・synced block — コンテンツ生成はLLMの仕事(設計原則3)で、人手の再利用部品が不要
- Notion AI相当 — 台帳ツールを持つ常駐チューターが既に上位互換
- ブラウザ的タブ管理 — Web版ではブラウザのタブが既にその機能。PWA/デスクトップ化(M4)後に再評価

## M3.5 推奨スコープ(この採否を反映した改訂版)

1. ライトテーマ移行(マップとナビ枠は濃紺維持)
2. サイドバー+ページツリー
3. トピック詳細ページ(ブロック構造+タスク化ボタン。core側は統合報告スキーマに practice_task を分離)
4. 概念ページ+バックリンク(星図クリックの遷移先)
5. ⌘K検索とチューター導線
6. KaTeX数式(依存追加はCodexが計画で提案)
7. タスクビュー強化
8. コントラストWCAG AAをテストで固定

順序の含意: M4のPWA化はこの画面構成の上に乗せる。
